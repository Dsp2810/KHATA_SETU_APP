    │             │          │             │          │             │
    └─────────────┘          └──────┬──────┘          └─────────────┘

                                   │ uses
                                   ▼
                            ┌─────────────┐
                            │ Repository  │
                            │             │
                            └─────────────┘

    ┌─────────────────────────────────────────────────────────────────────────┐
    │                        BLOC EXAMPLE FLOW                                 │
    │                                                                          │
    │  1. User taps "Add Customer" button                                     │
    │  2. UI dispatches AddCustomerEvent(customerData)                        │
    │  3. CustomerBloc receives event                                          │
    │  4. Bloc calls AddCustomerUseCase                                        │
    │  5. UseCase calls CustomerRepository.addCustomer()                       │
    │  6. Repository calls Remote/Local DataSource                            │
    │  7. Result flows back up                                                │
    │  8. Bloc emits CustomerAddedState or CustomerErrorState                 │
    │  9. UI rebuilds based on new state                                      │

## 🔐 Security Architecture

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                        SECURITY ARCHITECTURE                                 │
└─────────────────────────────────────────────────────────────────────────────┘

    ┌─────────────────────────────────────────────────────────────────────┐
    │                      AUTHENTICATION FLOW                             │
    │                                                                      │
    │   ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    │
    │   │  User    │    │  Server  │    │  JWT     │    │  Access  │    │
    │   │  Login   │───▶│  Verify  │───▶│  Token   │───▶│  Granted │    │
    │   └──────────┘    └──────────┘    └──────────┘    └──────────┘    │
    │                                                                      │
    └─────────────────────────────────────────────────────────────────────┘

    ┌─────────────────────────────────────────────────────────────────────┐
    │                      TOKEN MANAGEMENT                                │
    │                                                                      │
    │   ┌───────────────────────────────────────────────────────────────┐ │
    │   │                    ACCESS TOKEN                                │ │
    │   │                                                                │ │
    │   │   • Short-lived (15 minutes)                                  │ │
    │   │   • Contains: userId, shopId, role                            │ │
    │   │   • Stored in memory (Flutter)                                │ │
    │   │   • Sent in Authorization header                              │ │
    │   └───────────────────────────────────────────────────────────────┘ │
    │                                                                      │
    │   ┌───────────────────────────────────────────────────────────────┐ │
    │   │                   REFRESH TOKEN                                │ │
    │   │                                                                │ │
    │   │   • Long-lived (7 days)                                       │ │
    │   │   • Stored in secure storage                                  │ │
    │   │   • Used to get new access token                              │ │
    │   │   • Rotated on each use                                       │ │
    │   └───────────────────────────────────────────────────────────────┘ │
    │                                                                      │
    └─────────────────────────────────────────────────────────────────────┘

    ┌─────────────────────────────────────────────────────────────────────┐
    │                     ROLE-BASED ACCESS CONTROL                        │
    │                                                                      │
    │   ┌─────────────────┐  ┌─────────────────┐  ┌─────────────────┐    │
    │   │   SHOPKEEPER    │  │    CUSTOMER     │  │     ADMIN       │    │
    │   │                 │  │                 │  │                 │    │
    │   │ • Full access   │  │ • View ledger   │  │ • All access    │    │
    │   │ • CRUD all      │  │ • View balance  │  │ • User mgmt     │    │
    │   │ • Reports       │  │ • View receipts │  │ • Analytics     │    │
    │   │ • Settings      │  │                 │  │ • System config │    │
    │   └─────────────────┘  └─────────────────┘  └─────────────────┘    │
    │                                                                      │
    └─────────────────────────────────────────────────────────────────────┘

    ┌─────────────────────────────────────────────────────────────────────┐
    │                      SECURITY MEASURES                               │
    │                                                                      │
    │   1. Password Hashing           : Bcrypt with salt rounds (12)      │
    │   2. JWT Secret                 : Strong random secret key          │
    │   3. HTTPS                      : All API calls over TLS            │
    │   4. Rate Limiting             : 100 requests/minute per IP         │
    │   5. Input Validation          : Joi/express-validator              │
    │   6. SQL Injection Prevention  : Mongoose ORM                       │
    │   7. XSS Protection            : Helmet middleware                  │
    │   8. CORS                      : Restricted origins                 │
    │   9. Biometric Auth            : Device-level security              │
    │  10. Secure Storage            : flutter_secure_storage             │
    │                                                                      │
    └─────────────────────────────────────────────────────────────────────┘
```
